# Настройка работников

## Добавление роли "Работник"

После выполнения `supabase_setup.sql`, нужно обновить существующую таблицу:

```sql
-- Добавляем колонку is_worker если её нет
ALTER TABLE qr_auth_users 
ADD COLUMN IF NOT EXISTS is_worker BOOLEAN DEFAULT FALSE;
```

## Назначение работников

### Назначить конкретного пользователя работником:

```sql
UPDATE qr_auth_users 
SET is_worker = true 
WHERE telegram_id = TELEGRAM_ID_ПОЛЬЗОВАТЕЛЯ;
```

### Назначить всех работниками (для тестирования):

```sql
UPDATE qr_auth_users 
SET is_worker = true;
```

### Назначить админа (админ автоматически может сканировать):

```sql
UPDATE qr_auth_users 
SET is_admin = true, is_worker = true 
WHERE telegram_id = YOUR_TELEGRAM_ID;
```

## Как узнать telegram_id пользователя?

1. Пользователь открывает приложение
2. Нажимает на Debug (🐛) внизу справа
3. В логах видит свой ID
4. Отправляет ID админу
5. Админ выполняет SQL выше

## Генерация QR-кодов

Админ может генерировать QR-коды во вкладке "Генератор".

### Формат QR-кодов:

Все QR-коды имеют префикс `QR_AUTH_` для защиты.

Примеры:
- `ENTRANCE_1` → генерирует `QR_AUTH_ENTRANCE_1`
- `OFFICE_MAIN` → генерирует `QR_AUTH_OFFICE_MAIN`
- `ROOM_101` → генерирует `QR_AUTH_ROOM_101`

### Что вписывать в поле генератора:

- `ENTRANCE_1` - вход 1
- `ENTRANCE_2` - вход 2
- `OFFICE_MAIN` - главный офис
- `ROOM_101` - комната 101
- `CHECKPOINT_A` - контрольная точка А

Можно использовать любой текст, главное чтобы он был уникальным для каждой точки.

### Защита QR-кодов:

- Обычные QR-коды (без префикса `QR_AUTH_`) не будут сканироваться
- Только QR-коды созданные через генератор работают
- Это защищает от случайного сканирования чужих QR-кодов
