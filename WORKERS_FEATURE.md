# Разделение Пользователей и Работников

## Изменения

### 1. Исправлен шрифт в выпадающих днях
Добавлен класс `.day-detail-item` в список тонких шрифтов.

### 2. Добавлена вкладка "Работники"
Теперь есть две отдельные вкладки:

#### Пользователи
- Показывает только тех, кто НЕ является работником
- Функции:
  - Назначить работником
  - Удалить пользователя

#### Работники  
- Показывает только тех, у кого is_worker = true
- Функции:
  - Редактировать (имя и пол) - inline форма
  - Отнять статус (возвращает в пользователи)

### 3. Inline редактирование
Вместо модального окна - форма раскрывается прямо в карточке работника:
- Поле ввода имени
- Кнопки выбора пола (М/Ж)
- Кнопки "Отмена" и "Сохранить"

### 4. Новые поля в БД
Добавлены колонки:
- `display_name` - отображаемое имя (можно редактировать)
- `gender` - пол (M или F)

## SQL миграция

Выполните в Supabase SQL Editor:

```sql
-- Добавляем поля для редактирования работников
ALTER TABLE qr_auth_users 
ADD COLUMN IF NOT EXISTS display_name TEXT;

ALTER TABLE qr_auth_users 
ADD COLUMN IF NOT EXISTS gender CHAR(1) DEFAULT 'M';
```

Или используйте файл `add_worker_fields.sql`

## Логика работы

### Новый пользователь заходит в бота:
1. Попадает в таблицу `qr_auth_users` с `is_worker = false`
2. Админ видит его во вкладке "Пользователи"
3. Админ нажимает "Назначить работником"
4. Пользователь переходит во вкладку "Работники"

### Работник:
1. Отображается во вкладке "Работники"
2. Админ может:
   - Редактировать отображаемое имя
   - Изменить пол (для правильного склонения "Пришел/Пришла")
   - Отнять статус работника

### Отнять статус:
1. Админ нажимает "Отнять статус"
2. `is_worker` меняется на `false`
3. Бывший работник возвращается во вкладку "Пользователи"
4. Сохраняются его `display_name` и `gender` (на случай повторного назначения)
