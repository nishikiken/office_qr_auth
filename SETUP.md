# Инструкция по настройке QR Auth Mini App

## 1. Настройка Supabase

1. Используем существующий проект Supabase (уже настроен в коде)
2. Выполните SQL из файла `supabase_setup.sql` в SQL Editor вашего проекта
3. Таблицы будут называться `qr_auth_users` и `qr_auth_logs` (с префиксом чтобы не конфликтовать с другими приложениями)

## 2. Backend (опционально)

Backend не обязателен - приложение работает напрямую с Supabase через JavaScript.
Если нужен дополнительный функционал, можно использовать FastAPI backend из папки `backend/`.

## 3. Frontend

Frontend уже настроен на ваш существующий Supabase проект.
Просто задеплойте содержимое папки `frontend/` на любой хостинг.

## 4. Настройка Telegram Bot

1. Создайте бота через [@BotFather](https://t.me/BotFather)
2. Получите токен бота
3. Настройте Mini App:
   - `/newapp` - создать новое приложение
   - Выберите вашего бота
   - Укажите URL где размещен frontend (например, GitHub Pages или Vercel)
   - Загрузите иконку (опционально)

## 5. Назначение админа

Выполните SQL в Supabase SQL Editor:
```sql
UPDATE qr_auth_users 
SET is_admin = true 
WHERE telegram_id = YOUR_TELEGRAM_ID;
```

Чтобы узнать свой telegram_id:
1. Откройте приложение как обычный пользователь
2. Нажмите на кнопку Debug (🐛) внизу справа
3. Найдите свой ID в логах
4. Выполните SQL выше с вашим ID

## 6. Деплой Frontend

Варианты:
- GitHub Pages
- Vercel
- Netlify
- Cloudflare Pages

Просто загрузите содержимое папки `frontend/`

## Использование

- Обычные пользователи: открывают приложение → сканируют QR → авторизация отправляется
- Админы: открывают приложение → видят список всех авторизаций в реальном времени
